Log File

Depending on the target type, the TDMS log file can be found in the following locations:

  • Windows: "<System Definition Folder Path>\Logs\telemetry.tdms"

  • RT: "c:\ni-rt\NIVeriStand\Logs\telemetry.tdms"

By default, Overwrite File If Exists? is enabled. This option will overwrite the log file during each deployment of the system definition.

Note: If Overwrite File If Exists? is not enabled, the custom device will only create a new log file if one does not already exist on the target. If a log file does exist, the Error Code channel will report Error 6: Generic File I/O Error.

Disk Size Requirements

The resulting TDMS log file size depends on the system configuration and runtime. You can estimate the file size using the following formula:

  • File Size [bytes] = <TNC> * <Target Rate> [Hz] * <Running Time> [sec] * 8 [bytes]

where:

  • <TNC> represents the total number of logged channels. This number is the sum of 13 System Channels, 1 for Physical Memory usage, and the total number of target CPU cores running in the VeriStand system.
  • <Target Rate> represents the VeriStand system's run rate.
  • <Running Time> represents the deployed system's runtime.
  • 8 [bytes] represents the disk size for one double-precision, floating-point (DBL) numeric value. All logged channels from the file are DBL.

The following example is of a size calculation for a standard quad core system running at 1 [kHz] for a minute:

  • File Size [bytes] = 18 * 1000 * 60 * 8 = 8,640,000 [bytes] = 8.24 [MB]
